# 通信组网与边缘处理仿真

这是一个面向课程设计的单文件 HTML 仿真页面，用于展示多节点通信、链路拥塞、主备路径切换、节点故障隔离和优先级队列变化。界面采用克制的粉、蓝、绿三色主题，不使用外部图片资源。

## 在线或本地运行

- 本地：下载 `index.html`，使用 Chrome 或 Edge 打开。
- GitHub Pages：在仓库 `Settings → Pages` 中选择从 `main` 分支根目录发布。
- 页面不依赖服务器、数据库或第三方在线资源。

## 可演示场景

1. 正常运行：16 个节点在线，队列保持低水位。
2. 链路拥塞：共享带宽下降，低优先级数据先降频并进入缓存，之后解除拥塞并排空缓存。
3. 骨干断链：业务切换到邻近骨干节点的备份路径，主链路恢复后再切回。
4. GF7 离线：故障节点被隔离，其余节点继续运行，随后模拟节点重新接入。

## 仿真边界

页面从雷达前端输出点迹之后开始，重点验证通信、缓存和边缘调度。页面中的时延、吞吐、积压等数值均为合成仿真数据，不代表设备实测结果。

## 使用说明

点击页面顶部的场景按钮开始一次场景实验；页面会自动显示故障、处置和恢复过程。六项关键指标、固定量程趋势图、四场景基准卡和优先级队列会同步变化。“近5秒 P95 时延”表示约 95% 的近5秒已送达数据，其时延不超过该值。“暂停”和“重置”用于控制演示过程；“导出 CSV”可保存当前会话中的关键指标。
